# Day 8 - Lightning Web Components (LWC) Basics

## What is LWC?

Lightning Web Components (LWC) is Salesforce's modern UI framework used to build fast, reusable, and interactive user interfaces. LWC is based on modern web standards such as HTML, JavaScript, and CSS.

LWC helps developers create reusable components that can be used across multiple Salesforce applications.

---

## Why Salesforce Uses LWC

Salesforce introduced LWC to provide:

* Better performance
* Faster page loading
* Reusable components
* Modern web development experience
* Easy maintenance

LWC follows a component-based architecture that makes applications modular and scalable.

---

## Component-Based Architecture

A component is a reusable piece of UI that performs a specific task.

Example:

Student Dashboard can be divided into:

* Header Component
* Student Information Component
* Attendance Component
* Notification Component

Benefits:

* Reusability
* Easy maintenance
* Better scalability
* Cleaner code structure

## Frontend vs Backend Logic

### Frontend (UI)

* Button clicks
* Form display
* Showing notifications
* User interactions

### Backend (Apex)

* Fee calculations
* Data processing
* Database operations
* Complex validations

Keeping UI and business logic separate improves maintainability.

## Reflection

Modern enterprise systems use component-based architecture because it improves performance, maintainability, and scalability. Reusable components allow developers to build large applications efficiently while keeping the user experience consistent.
