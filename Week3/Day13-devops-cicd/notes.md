# Day 13 Notes

## Key Learnings

### DevOps

DevOps combines:

* Development
* Operations

Its goal is to improve software delivery through automation and collaboration.

---

### CI (Continuous Integration)

Developers regularly merge code into a shared repository.

Benefits:

* Early bug detection
* Better collaboration
* Faster development

---

### CD (Continuous Deployment/Delivery)

Automates deployment processes.

Benefits:

* Faster releases
* Reduced manual work
* Improved reliability

---

### GitHub Actions

GitHub Actions can automate:

* Testing
* Validation
* Deployment

This reduces human error.

---

### Why Production Should Be Protected

Production contains real users and real data.

Risks of direct changes:

* System outages
* Data corruption
* Broken workflows

Safe deployments reduce these risks.

---

### Team Collaboration

Large projects require:

* Version control
* Branches
* Code reviews
* Automated testing

Without these practices, projects become difficult to manage.

---

## Personal Reflection

Today I learned that software engineering is not only about writing code. DevOps practices, CI/CD pipelines, and deployment workflows help teams deliver reliable software while minimizing risks.
