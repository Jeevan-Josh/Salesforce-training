# Day 12 Notes

## Key Learnings

### Salesforce DX

Salesforce DX modernizes Salesforce development by treating metadata as source code.

Key Concepts:

* Source-driven development
* Scratch orgs
* Version control integration
* Team collaboration

---

### Salesforce CLI

CLI enables developers to interact with Salesforce using commands.

Common Uses:

* Project creation
* Authentication
* Metadata deployment
* Testing
* Automation

---

### GitHub

GitHub helps teams:

* Store code
* Track changes
* Collaborate efficiently
* Manage releases

---

### Enterprise Development vs College Projects

College Projects:

* Usually individual work
* Small scale
* Minimal testing

Enterprise Software:

* Large teams
* Strict testing
* Deployment pipelines
* Rollback strategies
* Continuous maintenance

---

### Importance of Workflow

Without workflow:

* Bugs increase
* Deployments fail
* Collaboration becomes difficult

With workflow:

* Better quality
* Faster releases
* Reliable systems

---

## Personal Reflection

Today I learned that enterprise software development requires planning, collaboration, testing, and deployment strategies. Tools like Salesforce DX, CLI, and GitHub help developers build and manage large applications efficiently.
